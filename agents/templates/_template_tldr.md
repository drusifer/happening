# Template for TL;DR summaries

## Too Long Don't Read 

When summarizing content for the CHAT.md or archive notes, use one of the the following forms:


### 1. The "Action-Oriented" Template
**Best for**: Internal emails, project updates, or professional memos where the reader needs to know if they need to do something.
```markdown
TL;DR: [One-sentence summary of the core news/update].
The Goal: [Why this matters]
Current Status: [Where we are now]
Action Required: [What the reader needs to do + Deadline]
```

### 2. The "Context-First" Template
**Best for**: Lengthy articles, research papers, or deep-dive blog posts.
```markdown
TL;DR: [Insert a 2-3 sentence summary that covers the Who, What, and Why].
Key Insight: [The most important discovery or takeaway].
Bottom Line: [The ultimate conclusion or "so what"].
```

### 3. The "BLUF" (Boring but Effective)

**Best for**: High-stakes corporate environments (Bottom Line Up Front).

```markdown
Summary: [Clear statement of the result or decision]. 
Impact: [How this affects the business/project/team]. 
Next Steps: [List 2-3 immediate milestones].
```

### 4. The "Technical/Feature" Template
**Best for**: Software release notes, documentation, or technical proposals.
```markdown
Overview: [Summary of the change/feature].
Problem: [What was broken or missing].
Solution: [What was implemented].
Breaking Changes: [Yes/No + brief detail].
```
